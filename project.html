Wine Classifier

As part of a Year 2 Coursework, My colleage and I were assigned a task to classify a set of wine samples into a certain number of classes. We ﬁrst extract discriminative features from the wine data samples, then built a classiﬁer with diffrent algorithms such as Naive Bayes, K-Nearest Neighbours and PCA that uses such features to recognise what class a given sample belongs to. Click the button on the left to read more about it :)

This dataset consists of 178 samples, each corresponding to a wine. Each wine is produced in the same region in Italy, and each wine derives from one of three diﬀerent cultivars (or type of plant). We will treat the three diﬀerent cultivars as classes. Each sample contains 13 features/dimensions, corresponding to a chemical constituent of the wine. The source code can be found here.

Sentiment Analysis

Ghyston, a bespoke software development company based in Bristol uses a range of open source tools including languages, libraries and frameworks, for developing software. The problem they face is determining which pieces of software to use. One metric used when deciding is their popularity, since this tells them if other users experiences with the software were positive or negative. Sites such as GitHub have existing ratings which can be used to rank the most starred repositories, but this does not tell the whole story. Click the button on the left to read more about it and try out our application :)

Me and three other students created techsentiment.com(Link has expired) to ease this problem. The application looks for posts on social media about a particular piece of software and extracts a sentiment value from them. An aggregate sentiment of the posts would be extremely useful in social media monitoring as it allows us to gain a better synopsis of the wider public opinion behind certain topics in software and provide an open source project's real value in the eyes of the developer community and help our client in choosing which software to use in their projects. Conway's Game of Life

My teammate and I designed and implemented a concurrent multi-threaded program which simulates British mathematician John Horton Conway's cellular automaton game named ‘The Game of Life’,on XMOS's xCore-200 Explorer board. The game was developed in a XC, a parallel programming language. Click the button on the left to read more about it :)

We divided the computation of different tasks and split the grid (based on the number of workers), by implementing a “message passing model “, through exchanging messages in different channels, to prevent easily parallelizable tasks. We used a Hard Channel to communicate between our distributor and our worker threads and a Soft Channel to exchange information between our LEDs, Buttons, Orientation, I/O streams to the distributor. Source code can be found here.

Kernel OS

The overarching goal of this concurrent programming assignment was to develop an initially simple but then increasingly capable operating system kernel using C and Assembly. It should execute and thus manage resources on a specific ARM-based target platform, namely a RealView Platform Baseboard for Cortex-A8 emulated by QEMU2. Click the button on the left to read more about it :)

The kernel developed was based on the concept of cooperative multitasking: driven by regular invocation of the yield system call, it context switched between and so concurrently executed a fixed number of user processes (that stemmed from statically linked user programs). A special-purpose scheduling algorithm was used for the process control block and Interprocess communication was used to solve the dining philosiphers problem. Source code can be found here.
